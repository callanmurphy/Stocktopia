a small js library to display stock charts and data.

Overview

  • This JS library can be used to manipulate the DOM to display stock data in an easy way
  • Stock objects can be created in the form Stock(ticker, data)
  • data must be sent as an object in the following format:

      • let x = ['10:00','11:00','12:00','13:00','14:00'];

      let y = [339.54, 342.33, 356.93, 324.21,298.25];

      let data = [

         { x: x[0], y: y[0]},

         { x: x[1], y: y[1]},

         { x: x[2], y: y[2]},

         { x: x[3], y: y[3]},

         { x: x[4], y: y[4]}

      ]

    Stock objects have two functions associated with them:

  • .makeChart()
  • .makeStats()

      • aapl = new Stock('AAPL', data);

      aapl.makeChart();

      aapl.makeStats();

    Concept

    While many JavaScript libraries exist to create and manipulate graphs of information, there are not many libraries which specifically allow for an easy DOM manipulation of stock data. Using a few function calls, a developer can select which types of data they would like presented on the DOM (i.e., volume, market cap, etc.) for a given ticker. Additionally, the data will have a few different CSS styles to be displayed in (such as different colours) to provide flexibility in the webpage appearance. This library will save developers the time of presenting stock data in an effective, consistent manner.

    The library will use a modern design that works well across different devices and screen sizes. For users, the consistency and organization of the stock data will allow them to easily find the financial information they need. Users will also be able to interact with the DOM, selecting specific time frames for the data.

    Use Cases

    For a news application, having the ability to easily display stock data would be effective for financial articles. As a developer, the data would be easy to represent and modify, with little work needed to organize the statistics into a news piece. Users would find the information useful and accessible, especially if a stock price has changed drastically since publishing. For instance, in an article titled “GameStop Stocks Fall 50%”, having live stock data would allow the user to see if the stock recovered since the time that the article was published.

    In an online stock brokerage, the developer has many other details to worry about than the representation of the data itself. Having a library to return stock ticker data could speed up the process of data implementation, so that developers can focus on customer orders, site navigation and overall design. Users would be able to search for and view stock data in a consistent manner, easily finding the information they need to determine which stocks to buy and sell.

    On the client application for a financial planning firm, presenting effective stock data would inform users how their investment portfolios are performing. While they would not be able to buy and sell shares like in a brokerage application, they could view their mutual funds or ETFs that they invested in with their advisor. The developer could include this data alongside information about why the firm chose specific funds, guiding the client to better understand the investment process.
